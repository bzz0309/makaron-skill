# Foreground Lyrics Video

这是一个跨 agent 可复用的创作工作流，用于制作“前景人物 + 后景动态歌词/音乐视觉层”的音乐感图片、视频或后期合成素材。它不是单一视频生成提示词，而是一套通用分层逻辑：人物永远是前景主体，歌词、文字、乐谱或音乐图形只作为背景氛围层，并按节奏持续运动或变化。

## STEP 1: 判断任务形态

先判断用户要的是哪种产物，再选择合适的 agent 组合：

- 单张视觉模板：输出“人物前景 + 歌词背景”的图片或封面，可交给 image agent。
- 单段视频：人物做听歌动作，背景动态变化，可交给 video agent。
- 多段素材：人物层、背景层、音乐层分开生成，再交给 editing agent 合成。
- 已有视频/图片改造：保留原人物或构图，只替换/增强背景动态歌词层。
- 带音乐成片：video agent 生成画面后，如无可用音频，再由 music agent 生成氛围音乐，editing agent 合成。

不要默认固定某个模型、某个路径或某种实现方式。具体模型由运行环境和上游调度决定。

## STEP 2: 锁定核心层级

所有 agent 都必须遵循以下层级，不可反转：

```text
前景层：人物主体，清晰、居中或明确占主体
动作层：戴耳机、调整耳机、闭眼听歌、轻微点头、轻微左右摆动
背景层：英文歌词、音乐文字、乐谱、声波或抽象音乐视觉
运动层：背景文字持续跑动、滚动、滑动、重组，并每隔数秒变化
声音层：可选的音乐氛围、节拍或最终混音
```

关键约束：

- 人物必须在前景，不能被背景文字替代。
- 背景是动态氛围层，不是主角。
- 歌词/文字不能遮挡人物脸部、眼睛、耳机和主要轮廓。
- 动作要像听歌时的沉浸律动，不要大幅跳舞，除非用户明确要求。
- 背景每约 3–4 秒变化一次排版、运动方向、密度或视觉节奏。
- 不做纯文字墙，不做普通字幕视频，不做花哨转场堆叠。

## STEP 3: 输入分析规范

当用户提供参考图、参考视频或人设时，先提取可迁移的设计信息。

人物信息：

- 人物位置：居中、偏左、偏右、半身、中特写、全身。
- 画面占比：推荐人物高度约 70%–80%，但应服从用户参考。
- 姿态表情：闭眼、低头、侧头、放松、沉浸、轻微律动。
- 听歌符号：头戴耳机、耳机线、手扶耳机、播放器、麦克风。
- 身份一致：若用户要求保留人物，必须强调脸部、发型、服装轮廓一致。

背景信息：

- 原背景是否为乐谱、歌词、文字墙、屏幕、投影、霓虹、抽象声波。
- 背景与人物是平面墙关系、屏幕后景关系，还是空间景深关系。
- 背景变化是硬切、滑动、滚动、缩放、重组，还是光影渐变。
- 背景是否需要黑白、低饱和、复古胶片、极简、赛博或舞台感。

如果参考视频语义分析失败，可以抽帧或让用户补充描述，但不得把需求误解为纯文字背景。

## STEP 4: 背景歌词与文字设计

优先使用原创英文短句或用户提供的授权歌词。没有授权时，不要直接复制真实歌曲歌词。

可复用原创短句类型：

- 情绪型：`drown in the music`, `lost in the rhythm`, `echoes in my head`
- 音乐型：`heartbeat on repeat`, `silence turns to sound`, `let the music breathe`
- 氛围型：`falling into waves`, `feel the night move`, `running through echoes`
- 抽象型：`slow waves`, `deep bass`, `soft noise`, `midnight signal`

背景运动模板：

- 横向跑动：多行小字从左到右或右到左移动，适合作为底层纹理。
- 纵向滚动：大字号短句从下往上或上往下滑动，适合每段变化点。
- 分层错速：远处小字慢速、近处大字快速掠过，但不得遮脸。
- 重组切换：每 3–4 秒重新排版，从稀疏变密集或从大字变多行。
- 投影墙：文字像投在人物身后的墙面或屏幕上，有轻微运动模糊。

## STEP 5: Agent 交接方式

### 给 image agent

用于生成人物设定图、封面图、背景板或首帧。任务描述应包含：

- 前景人物主体与姿态。
- 是否戴耳机或做听歌动作。
- 背景为音乐文字/歌词视觉层。
- 文字只作为背景，不遮挡脸。
- 输出比例和风格。

不要要求 image agent 生成视频运动；只让它产出稳定的视觉基准或分层素材。

### 给 video agent

用于生成单段视频、图生视频或参考视频复刻。任务描述应包含：

- 前景人物动作：戴耳机、调整耳机、闭眼、轻微点头或摆动。
- 背景运动：歌词持续跑动、滚动、滑动、重组。
- 变化节奏：每 3–4 秒改变一次背景排版或方向。
- 遮挡约束：文字不得盖住人物脸。
- 风格约束：极简、音乐感、低饱和或用户指定风格。

如果模型不能稳定生成文字，可把背景描述为“英文歌词感文字块、音乐排版、抽象英文短句”，保留视觉逻辑。

### 给 music agent

用于补充无版权音乐或氛围 BGM。任务描述应包含：

- 音乐情绪：lo-fi、chill、R&B、ambient pop、dream pop 等。
- 节奏需求：适合人物轻微摆动，不要过快。
- 是否需要歌词：默认不需要，因为画面背景已有文字；用户明确要求歌曲时再生成带歌词音乐。
- 时长不承诺精确，由后期裁切或循环适配。

### 给 editing agent

用于合成分层素材、加入音乐、调色、裁切、字幕或最终成片。任务描述应包含：

- 将人物视频/图片作为前景主体。
- 将动态歌词背景放在后景，不遮挡脸。
- 加入音乐并保持人物律动与背景变化协调。
- 需要时做黑白、低饱和、复古胶片或统一色调。
- 若有旁白或音乐，按音频层级混音，避免背景音乐压过人声。

不要在上游硬编码具体 ffmpeg 滤镜、字体像素或实现细节，把实现交给 editing agent。

## STEP 6: 默认视频节奏

当用户没有给时长和分镜时，可采用 10–12 秒结构：

- 0–3 秒：人物进入听歌状态，调整耳机或闭眼；背景小字慢速横向跑动。
- 3–6 秒：人物轻微左右摆动；背景切换为大字号短句纵向滚动。
- 6–9 秒：镜头轻微推进或保持稳定；背景多层文字错速滑动并重组。
- 9–12 秒：人物保持沉浸姿态；背景运动收束或变稀疏，自然结束。

如果用户提供音乐，应按音乐节拍和段落重新安排变化点，不机械套用固定秒数。

## STEP 7: 质量检查与纠偏

生成或合成后检查：

- 人物是否始终是前景主体。
- 是否有戴耳机、调整耳机、听歌或随音乐轻微摆动的动作。
- 背景是否是动态歌词/音乐视觉层，而不是静态图案。
- 背景是否每隔几秒变化，而不是一直不动。
- 文字是否没有遮住人物脸。
- 输出是否不是纯文字墙。
- 音乐与人物动作、背景变化是否协调。

常见失败与修正：

- 变成纯文字墙：强化“人物占主体，背景文字只在后方”。
- 人物太小：强化“人物高度约 70%–80%，居中中景/中特写”。
- 文字遮脸：强化“脸部安全区，文字避开眼睛、鼻子、嘴和耳机主体”。
- 背景不动：强化“文字持续跑动、滚动、滑动，每 3–4 秒变化”。
- 动作太夸张：强化“小幅摆动、轻微点头、沉浸听歌，不跳舞”。
- 文字不可读或乱码：接受抽象英文文字块作为视觉层，或改由后期叠加可控文字。

## STEP 8: 通用化原则

这个 Skill 不绑定任何具体项目素材、具体模型、固定歌词或单一路径。使用时应把用户当轮提供的图片、视频、音乐、人设和偏好作为输入；如果没有输入，则使用通用默认。所有生成结果应保存在当前会话项目中，并把可复用的中间资产路径传递给下一步 agent。
